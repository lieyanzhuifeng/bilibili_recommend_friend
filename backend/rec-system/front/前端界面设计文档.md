# 前端界面设计文档

## 1. 项目概述

本项目是一个基于Vue.js开发的社交类应用，主要功能包括用户登录、用户推荐、好友管理和聊天功能。界面设计风格简洁现代，采用了类似哔哩哔哩的设计语言，具有良好的用户体验和交互效果。

## 2. 页面设计与功能分析

### 2.1 登录页面 (Login.vue)

#### UI设计
- **整体布局**: 居中的登录框，采用渐变色背景，视觉上简洁明快
- **Logo区域**: 显示"哔哩哔哩"和"年轻人的潮流文化社区"文字，突出品牌特色
- **登录表单**: 包含用户ID输入框和登录按钮，输入框支持回车键触发登录
- **提示信息**: 底部显示"提示：输入用户ID即可登录"，增强用户引导

#### 功能
- 用户ID输入与验证
- 登录状态管理
- 登录成功后的页面跳转

#### 流程
1. 用户访问登录页面
2. 输入用户ID
3. 点击登录按钮或按回车键
4. 系统验证用户ID
5. 验证成功后跳转到个人资料页面
6. 验证失败则显示错误提示

### 2.2 用户推荐系统 (Recommendations.vue)

#### UI设计
- **整体布局**: 左侧边栏 + 右侧主内容区的经典布局
- **侧边栏**: 包含筛选条件，如用户活跃度、夜猫子程度等，使用复选框和标签
- **主内容区**: 顶部为推荐API选择按钮，中间为筛选API选择按钮，下方为筛选条件输入区域
- **响应式设计**: 页面元素根据屏幕大小自适应调整

#### 功能
- 推荐API选择
- 筛选条件设置（用户活跃度、夜猫子程度）
- 多维度筛选（UP主、标签、视频、系列）
- 搜索功能（支持防抖搜索）
- 筛选结果展示

#### 流程
1. 用户进入推荐页面
2. 选择推荐API类型
3. 选择筛选API类型
4. 设置具体筛选条件
5. 输入关键词进行搜索
6. 选择搜索结果
7. 系统根据筛选条件生成推荐结果

### 2.3 好友列表页面 (Friends.vue)

#### UI设计
- **整体布局**: 顶部标题栏 + 标签切换 + 好友列表
- **标题栏**: 显示"我的好友"和"添加好友"按钮
- **标签切换**: 包含"全部好友"、"在线好友"、"未读消息"三个标签
- **好友列表**: 使用卡片组件展示好友信息，包含头像、用户名、简介、在线状态等
- **空状态提示**: 当没有好友时显示提示信息和添加好友按钮

#### 功能
- 好友分类展示（全部、在线、未读消息）
- 好友搜索功能
- 添加好友功能
- 好友聊天入口
- 好友状态显示

#### 流程
1. 用户进入好友列表页面
2. 选择好友分类标签
3. 浏览好友列表
4. 点击好友卡片进入聊天页面
5. 点击"添加好友"按钮打开添加好友弹窗
6. 输入用户ID搜索并添加好友

### 2.4 聊天页面 (Chat.vue)

#### UI设计
- **整体布局**: 左侧好友列表 + 右侧聊天窗口
- **聊天窗口**: 包含聊天头部、消息区域、输入区域
- **聊天头部**: 显示联系人信息（头像、姓名、在线状态）和操作按钮
- **消息区域**: 区分发送和接收的消息，支持消息状态显示
- **输入区域**: 包含消息输入框和发送按钮

#### 功能
- 好友聊天功能
- 消息发送与接收
- 消息状态显示（发送中、已发送、已读）
- 好友在线状态显示
- 聊天历史记录查看
- 消息搜索功能
- 聊天操作（清空聊天、删除聊天记录、屏蔽用户）

#### 流程
1. 用户从好友列表选择一个好友开始聊天
2. 输入消息内容
3. 点击发送按钮或按回车键发送消息
4. 系统将消息发送到服务器并更新本地聊天记录
5. 接收好友发送的消息并实时更新
6. 可以查看聊天历史记录、搜索消息或进行其他聊天操作

### 2.5 好友管理页面 (FriendManagement.vue)

#### UI设计
- **整体布局**: 顶部标题 + 标签切换 + 内容区域
- **标签切换**: 包含"好友申请"和"我的好友"两个标签
- **好友申请**: 列表展示好友申请信息，包含头像、用户名、UID、申请时间和操作按钮
- **我的好友**: 列表展示所有好友信息，包含头像、用户名、UID和操作按钮

#### 功能
- 好友申请管理（接受、拒绝）
- 好友列表管理
- 好友搜索功能
- 好友删除功能

#### 流程
1. 用户进入好友管理页面
2. 选择"好友申请"标签查看新的好友申请
3. 对好友申请进行接受或拒绝操作
4. 选择"我的好友"标签查看所有好友
5. 可以搜索好友或进行删除操作

### 2.6 个人资料页面 (Profile.vue)

#### UI设计
- **整体布局**: 顶部个人信息卡片 + 内容导航 + 主要内容区
- **个人信息卡片**: 显示头像、用户名、UID、关注数、粉丝数、等级等信息
- **内容导航**: 包含"个人画像"等标签
- **主要内容区**: 左侧边栏显示基本信息，右侧显示个人画像内容

#### 功能
- 个人信息展示
- 个人画像查看
- 消息通知查看
- 资料编辑入口

#### 流程
1. 用户进入个人资料页面
2. 查看个人基本信息和统计数据
3. 选择"个人画像"标签查看分类分布
4. 点击"编辑资料"按钮进入编辑页面
5. 点击消息通知图标查看未读消息

### 2.7 编辑个人信息页面 (UserInfo.vue)

#### UI设计
- **整体布局**: 顶部标题 + 表单卡片
- **表单结构**: 分"基本信息"、"联系信息"、"偏好设置"等章节
- **表单元素**: 包含头像上传、用户名、简介、邮箱、手机号码等输入框
- **验证反馈**: 实时显示表单验证错误和字符计数

#### 功能
- 个人信息编辑
- 头像上传功能
- 表单验证
- 数据提交与保存

#### 流程
1. 用户进入编辑个人信息页面
2. 修改需要更新的信息（如头像、用户名、简介等）
3. 系统实时验证输入内容
4. 点击保存按钮提交修改
5. 提交成功后显示成功提示并返回个人资料页面
6. 提交失败则显示错误信息

## 3. 组件设计

### 3.1 Button.vue

#### UI设计
- **样式变体**: 支持primary、secondary、outline、text四种类型
- **尺寸变体**: 支持small、medium、large三种尺寸
- **状态样式**: 支持disabled、loading状态
- **图标支持**: 可添加图标前置显示

#### 功能
- 按钮点击事件处理
- 加载状态管理
- 禁用状态管理

### 3.2 Card.vue

#### UI设计
- **卡片结构**: 包含header、body、footer三个部分
- **阴影效果**: 支持阴影显示和hover效果
- **响应式设计**: 适应不同屏幕尺寸

#### 功能
- 内容容器组件
- 标题和页脚支持
- 可定制阴影效果

### 3.3 Navbar.vue

#### UI设计
- **固定顶部**: 导航栏固定在页面顶部
- **品牌标识**: 左侧显示品牌Logo
- **导航菜单**: 中间显示主要导航链接
- **用户信息**: 右侧显示用户头像和下拉菜单

#### 功能
- 页面导航
- 用户状态显示
- 下拉菜单功能

## 4. 整体设计风格

### 4.1 色彩方案
- **主色调**: 粉色系（#FB7299），与哔哩哔哩品牌一致
- **辅助色**: 蓝色（#3E97FF）用于secondary按钮
- **背景色**: 浅灰色（#f8f9fa）作为页面背景
- **文字色**: 深灰色（#333）作为主要文字颜色

### 4.2 排版
- **字体**: 使用系统默认字体栈，确保跨平台一致性
- **字号**: 建立清晰的字号层级，从标题到正文
- **行高**: 合理的行高设置，提高文本可读性

### 4.3 交互设计
- **动画效果**: 按钮hover效果、卡片阴影变化
- **反馈机制**: 表单验证反馈、操作成功/失败提示
- **加载状态**: 使用加载指示器显示操作进度
- **导航体验**: 清晰的导航结构，支持面包屑和标签页

## 5. 技术实现

### 5.1 技术栈
- **前端框架**: Vue.js 3
- **路由管理**: Vue Router
- **状态管理**: Pinia
- **UI组件**: 自定义组件（Button、Card、Navbar）
- **网络请求**: Axios

### 5.2 布局方案
- **响应式设计**: 使用CSS Flexbox和Grid实现响应式布局
- **组件化开发**: 将界面拆分为可复用的组件
- **布局模式**: 采用侧边栏 + 主内容区的经典布局模式

### 5.3 性能优化
- **组件懒加载**: 实现路由组件的懒加载
- **防抖搜索**: 减少搜索请求次数
- **图片优化**: 合理使用图片尺寸和格式
- **状态管理优化**: 合理使用Pinia进行状态管理

## 6. 总结

本项目的前端界面设计遵循了现代UI设计原则，具有良好的视觉效果和用户体验。各个页面的功能清晰明确，流程合理，交互流畅。通过组件化开发和响应式设计，确保了代码的可维护性和跨平台兼容性。整体设计风格统一，符合哔哩哔哩的品牌特色，为用户提供了一个友好、高效的社交应用体验。
